        <h2>Exercise 0: Answer the "Big Three"</h2>
        <p>For a recent data project, or for an existing data workflow, answer the “Big Three” post-mortem questions.</p>
        <ul>
            <li>What worked that you’d absolutely do again without change?</li>
            <li>What didn’t work that you would never do again?</li>
            <li>What worked, but you would do again different? How?</li>
        </ul>
    </div>
    <aside class="notes">
    Making sure that everyone comes to the postmortem with their thoughts in order, and structured around similar questions is important to ensure that everyone is working from the same playbook.
    <br><br>
    There are many subjective metrics that you can gather from participants (in addition to the broader quantitative metrics of the project that can help frame the process), but it essentially boils down to three big questions:
    </aside>

<section>
    <div class="full">
        <h2>Quantitative is easier to measure, but hard to filter</h2>
        <ul>
            <li>Goals met?</li>
            <li>Numbers changed?</li>
            <li>Graphs well</li>
        </ul>
    </div>
    <aside class="notes">
    The pre-meeting questionnaire is great qualitative feedback, but making sure that you have measurable objectives of a data project is part of general best practices. 
    <br><br>
    Making sure that you have gathered the results of the project itself can be great validation (or contrast) for the qualitative objectives. 
    </aside>
